# Linux 银行管理系统

基于 Linux System V 消息队列的银行管理系统，采用 C/S 架构，服务端多进程（fork）并发处理开户、存款、取款、转账四类业务。

## 项目结构

```
.
├── bank.h          # 公共头文件（消息结构体、操作类型、队列键值）
├── server.c        # 服务端（多进程 + 消息队列）
├── client.c        # 客户端（菜单交互 + 消息收发）
├── Makefile        # 工程编译管理
└── README.md
```

## 架构设计

```
┌──────────────────┐         ┌──────────────────────────────────┐
│     client       │         │            server                │
│                  │  req    │                                  │
│  菜单 → 组装消息 ──┼──msg──→ │  fork                            │
│                  │ 队列    │  ├── 子进程1: open_func  (开户)    │
│  显示结果 ← 解析  │  res    │  ├── 子进程2: save_func  (存款)    │
│                  │←─msg─── │  ├── 子进程3: take_func  (取款)    │
│                  │ 队列    │  └── 子进程4: trans_func (转账)    │
└──────────────────┘         └──────────────────────────────────┘
```

- **请求消息队列** (key=1234)：客户端发送业务请求
- **响应消息队列** (key=5678)：服务端返回处理结果
- 服务端 4 个子进程各自阻塞等待对应 `mtype` 的消息，互不干扰

## 消息协议

```c
typedef struct {
    long mtype;          // 消息类型 (1=开户, 2=存款, 3=取款, 4=转账)
    int opt;             // 操作类型
    char name[20];       // 姓名
    char pwd[20];        // 密码
    int account;         // 账号
    float money;         // 金额
    int to_account;      // 目标账号（转账）
    char result[128];    // 服务端返回结果
} MsgBuf;
```

## 快速开始

### 编译

```bash
make          # 编译 server 和 client
make clean    # 清理编译产物
```

### 运行

先启动服务端，再启动客户端：

```bash
# 终端1 - 启动服务端
./server &

# 终端2 - 启动客户端
./client
```

## 数据持久化

- `account_id.txt` — 记录当前最大账号 ID，开户时自增
- `account_<id>.txt` — 每个账户独立文件，存储格式：
  ```
  账号
  姓名
  密码
  余额
  ```

## 技术要点

- **进程管理**：`fork()` 创建子进程，各进程独立运行
- **IPC 通信**：System V 消息队列（`msgget` / `msgsnd` / `msgrcv`）
- **文件 IO**：C 标准库 `fopen` / `fscanf` / `fprintf` 持久化账户数据
- **编译管理**：Makefile 工程化编译，`-Wall -Wextra` 严格警告

## 清理消息队列

程序结束后，消息队列可能残留。手动清理：

```bash
ipcs -q                    # 查看消息队列
ipcrm -Q 0x000004d2       # 删除 key=1234 的队列
ipcrm -Q 0x0000162e       # 删除 key=5678 的队列
```
